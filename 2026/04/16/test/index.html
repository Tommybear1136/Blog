Transformers are RNNs: Fast Autoregressive Transformers with Linear Attention 論文詳解

Authors: Angelos Katharopoulos, Apoorv Vyas, Nikolaos Pappas, Francois Fleuret
Venue: ICML 2020
Paper Link, Code Link

Core contribution

  • 提出一個計算attention的方法，這個方法不會對q,k,v有任何constrain，不僅可以在training的時候讓運算的time complexity為O(N)、memory consumption為O(N)，同時在inference的時候，computational cost和memory consumption每個time step甚至永遠維持constant，不會受到input sequence或預測出的token增長而影響。

Related work

Efficient Transformers

The bottleneck is mainly caused by the global receptive field of self-attention, which processes contexts of N inputs with a quadratic memory and time complexity O().

  • 目前主要的解決方法有以下兩種:
    • Improve memory efficiency: 減少模型內部weight的大小 (利用weight pruning,weight factorization, weight quantization or knowledge distillation)，這樣做可以加速training和inference時間，但是因為機制不變，這樣做的time complexity依然為O()，仍然很難應用到long sequence上面。
    • Increase the context of self-attention: [1],[2] 和原始模型的asymptotic complexity仍然是一樣的。[3],[4] 和本篇論文比較像都有improve time complexity，但他們都對模型做了某種constrain導致在inference時無法apply，因此在 inference時time complexity和memory consumption依然是平方級遞增。
      • [1] 透過maintain previous contexts in memory，減少computational cost
      • [2] learns the optimal attention span per attention head，控制memory和computational cost
      • [3] introduces sparse factorizations of the attention matrix reducing the overall complexity to O()
      • [4] reduces complexity to O() by using locality-sensitive hashing (LSH) to perform fewer dot products.

Understanding Self-Attention

  • Kernel function:[5] considers attention as applying a kernel smoother over the inputs with the kernel scores being the similarity between inputs來試著理解attention的運作，而本篇則是利用kernel function的formulation來幫助加速attention運算。
  • 模型比喻: [6] provided theoretical proofs and empirical evidence that a multi-head selfattention with sufficient number of heads can express any convolutional layer，而本篇則是把self-attention layer看成RNN，讓我們在inference的時後可以更快。

Linearized softmax

For many years, softmax has been the bottleneck for training classification models with a large number of categories. 而最近有一些work[7],[8]試著去approximate softmax with a linear dot product of feature maps through sampling，本篇也是把softmax用linear dot product of feature maps來替代。

Transformers are RNNs

從上面的演算法和公式12我們可以把transformer想像成RNN，只是我的RNN現在有兩個hidden states，一個是分母的attention memory ，另一個是分子的normalizer memory ，然後每一個time step我會去update我的這兩個internal memory然後預測出對應的output token。從這個角度看你會發現這篇論文集合了transformer可以平行化訓練的優點，同時在inference的時候結合了RNN的優點可以只存儲兩個internal states而不是所有token的KV，讓模型的cost and memory for one prediction可以保持constant。

Key Experiments

主要是實驗在image generation和audio recognition的任務上，因為這些任務的sequence比較長，比較對象則都是和原始pytorch transformer以及Reformer[4]比較performance和complexity

實驗一

  • 可以看到Linear transformer會和softmax attention converge到同個位置，比lsh好
  • 再來是展示training的時候，Linear transformer forward/backward pass的time complexity和memory consumption會隨著training sequence length增加而線性上升，值得注意的是Reformer O()因為夠小所以在training也是非常不錯的線性遞增

實驗二

  • 在MNIST上performance和原始transformer差不多，但inference速度是300多倍
  • 在CIFAR-10 sequence更長的情況下，performance比原始transformer好，且inference速度是原本的4000多倍
